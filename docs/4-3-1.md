### Notes

- If the players still do not render, test the SVG standalone in a browser to confirm it works.
- Ensure the Markdown viewer supports inline SVG rendering. Some viewers may block certain SVG features.
2. Replace the inline SVG with a placeholder like `<!-- svg-inline: ../diagrams/soccer-pitch.svg -->`.
3. During the build, replace the placeholder with the SVG content. This keeps your source Markdown clean while generating the final output with inline SVGs.
- If you prefer not to change workspace trust, use one of these:
  - Open the SVG file directly in VSCode or a browser (to confirm it's valid).
  - Install an extension like "Markdown Preview Enhanced" or serve the docs with Live Server and view in a browser.
  - Use a build step to inline SVG into the Markdown before previewing.

Quick inline build (Node one-liner)
- From the repository root you can generate an inlined preview file (creates docs/4-3-1.inlined.md):
  node -e "const fs=require('fs');let md=fs.readFileSync('docs/4-3-1.md','utf8');md=md.replace(/<!--\s*svg-inline:\s*(.+?)\s*-->/g,(m,p)=>fs.readFileSync(p,'utf8'));fs.writeFileSync('docs/4-3-1.inlined.md',md);console.log('Wrote docs/4-3-1.inlined.md');"

How to use the inlined file
- Open docs/4-3-1.inlined.md in VSCode and preview. The external SVG will be embedded so VSCode's sanitizer won't attempt to load an external file.
- When you finish, delete the generated inlined file or add it to a build artifact folder.

If you still see a broken-image icon:
- Verify the exact filename and capitalization in ../diagrams/.
- Check the Developer Tools console in VSCode (Help → Toggle Developer Tools) for blocked loads or errors.
